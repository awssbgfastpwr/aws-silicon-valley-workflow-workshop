# Session 03: Cloud Access

AWS account access, IAM basics, and cloud safety setup for workshop participants.

## Focus

- AWS account sign-in flow
- IAM users, roles, and least privilege
- Billing alerts and budget awareness
- Secure credential handling
- Preparing for deployment sessions

## Safety Notes

Do not commit AWS access keys, secret keys, session tokens, `.env` files, or downloaded credential files to this repository.

## Lab Task

- Sign in with the access method approved by the workshop team.
- Identify the current AWS account and region.
- Review IAM users, roles, and least-privilege basics.
- Confirm billing alerts or budget visibility before deployment work.

## Done When

- You can explain the difference between an IAM user and an IAM role.
- You know which AWS account and region the workshop uses.
- You know where to check billing.
- You understand why credentials must stay out of GitHub.
